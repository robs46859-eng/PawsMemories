# Admin Access Map

Every module and feature reachable by an **admin** account, and how to get there.

Admin status comes from `users.is_admin`, surfaced to the client as
`userProfile.isAdmin` and enforced server-side by `isUserAdmin(phone)`.
The admin account for this deployment is **robs46859@gmail.com**.

> **Two independent gates.** The UI hides admin controls when `isAdmin` is false,
> and every admin API re-checks `isUserAdmin()` server-side. The UI gate is
> convenience; the server gate is the security boundary. Never rely on hiding a
> button alone — a hidden button is still a reachable endpoint.

---

## 1. Getting in

1. Sign in normally at `https://pawsome3d.com`
2. Admin surfaces appear automatically — there is no separate admin login or
   subdomain
3. Admin-only entries live in the **header overflow menu** (the `⋯` icon, far
   right of the top bar), below the standard items

## 3. Marketplace admin — what you can do

Reached via `/admin/marketplace`. This is the full catalog manager.

| Action | UI | API |
|---|---|---|
| List all listings with counts | Table view | `GET /api/admin/marketplace/listings` |
| Create a listing | *New listing* | `POST /api/admin/marketplace/listings` |
| Edit a listing | Click a row | `PATCH /api/admin/marketplace/listings/:id` |
| Reorder listings | Drag rows | `POST /api/admin/marketplace/listings/:id/reorder` |
| Upload a model or preview | *Upload* → file picker | `POST /api/admin/marketplace/upload-url` then direct PUT to Backblaze |
| Register an uploaded asset | Automatic after upload | `POST /api/admin/marketplace/assets` |
| Activate / retire an asset | Asset row controls | `PATCH /api/admin/marketplace/assets/:id` |
| View listing previews | Listing detail | `GET /api/admin/marketplace/listings/:id/previews` |
| View listing assets | Listing detail | `GET /api/admin/marketplace/listings/:id/assets` |

**Upload flow.** The browser never sends the file through your server. It asks
for a presigned URL, PUTs the bytes straight to Backblaze, then registers the
asset. `headPrivateObject()` re-reads what actually landed — the client's claimed
size and MIME type are never trusted.

**Storage split.** Source GLBs go to the **private** bucket
(`MEDIA_PRIVATE_BUCKET_NAME`) and are only ever served as short-lived presigned
URLs after an entitlement check. Previews go to the **public** bucket. This split
is the entire access-control model, because Backblaze has no per-object ACLs —
see the header comment in `storage.private.ts`.

## 4. Wags admin — monthly box review

Reached via `/admin/wags`.

Boxes are generated by Gemini on the 1st at 02:00 UTC and land at
`status = 'pending_review'`. **Nothing ships without you.**

| Action | Effect |
|---|---|
| Approve box | `status = 'approved'` → delivery proceeds |
| Reject box | `status = 'rejected'` → subscriber notified, month skipped |
| Swap an item | Sets `swapped_by_admin = 1` on that line item |
| Add admin notes | Stored on the box for audit |

API: `GET /api/admin/wags/boxes`, `PATCH /api/admin/wags/boxes/:id`,
`POST /api/admin/wags/boxes/:subscription`.

## 5. Admin privilege inside normal features

Admin also changes behaviour in ordinary screens rather than unlocking new ones:

| Where | Difference for admins |
|---|---|
| Model cap | Non-admins capped at `MODEL_CAP` (default 5) models; admins uncapped (`server.ts:3703`) |
| Paid-endpoint rate limits | Admin checks at `server.ts:4190`, `4962`, `5807`, `5898`, `5999`, `6357`, `6489` relax per-user quotas |
| Credit costs | Several generation paths check `isAdmin` before charging |
| Community moderation | `server.ts:2891` gates moderation actions |

## 9. Things with no admin UI yet

Worth knowing, because they need direct DB or API access:

- **Refund review** — `server/refunds.ts` mounts `/api/refunds` and
  `/api/admin/refunds`, but there is no admin screen
- **User management** — no UI for granting admin, adjusting credits, or
  suspending accounts; all direct SQL
- **Texture job inspection** — `texture_jobs.stats_json` holds the UV8 likeness
  report (`before`/`after`/`delta`/`improved`) but is only visible via SQL
