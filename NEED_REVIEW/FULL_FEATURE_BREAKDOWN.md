## 1. Product Overview

A browser-based **Building Information Model builder** inside PawsMemories/Pawsome3D. Users can (a) **author** a scaled building from parametric elements in a 3D viewport, (b) **import** real IFC files (IFC2X3 / IFC4 / IFC4X3) with full semantics, and (c) purchase **verified builds** ‚Äî a visual GLB "Shell" (60 cr) or a full IFC4 BIM package (300 cr) ‚Äî with a free pre-build gate and a paid post-build verification gate. Entry point: `BimModelBuilder.tsx` modal launched from `AvatarDashboard.tsx`.

## 2. Spatial Foundation (Phases 0‚Äì1)

### 2.1 Authoritative spatial metadata ‚Äî `src/three/spatial/types.ts` (306 lines, new)
- `ModelSpatialMetadataSchema` (zod-validated, versioned) records for any asset: **source unit** (enum of supported length units with `UNIT_TO_METERS` factors), **canonical bounds** (computed through nested node world transforms, not just root), **physical vs display scale**, **axes** (`UpAxis` Y/Z/X, `ForwardAxis` ¬±X/¬±Y/¬±Z, `Handedness` right/left), **accuracy class** (`ACCURACY_CLASSES` enum), **calibration method** (`CALIBRATION_METHODS` enum ‚Äî how the scale was established), **source hash** for provenance/lineage, and creation timestamp.
- `createSpatialMetadata()` factory enforces invariants at construction.
- Persisted in MySQL: `db.ts` adds spatial-metadata CRUD keyed by **tenant + asset kind + asset ID** (lines ~713, ~2626).
- Also threaded into the Animator: `src/animator/types.ts` `AssetMetadata.spatialMetadata` (sourceUnit, metersPerSourceUnit, canonical bounds, physicalScale, displayScale, accuracyClass, calibrationMethod, sourceHash).

### 2.2 Trusted physical scale
- Assets carrying trusted spatial metadata **bypass the longest-edge auto-fit** in the browser viewer and AR (`ObjectModel.tsx`, `eighthWallAR.ts` changes): a 10 m building renders 10 m, never normalized to fit the view. Display fitting can no longer silently change physical scale.

### 2.3 Measurement utilities ‚Äî `src/three/spatial/measurement.ts` (140 lines, new)
- Display units: `m, cm, mm, ft, in, ft/in` (feet + fractional-inch composite) via `DISPLAY_FACTORS`.
- `pointDistance` (3D euclidean), `boundsExtent`, `boundsCenter`, axis-aligned dimension readouts, unit conversion for display ‚Äî all pure and unit-tested.

### 2.4 Fixture & convention system (Phase 0 baseline)
- Reproducible GLB fixtures: `100mm-cube.glb`, `1m-cube.glb`, `10m-cube.glb` (generated by `scripts/generate-bim-fixtures.mjs`) with `glb-manifest.json` recording known bounds.
- IFC fixtures generated by API (`scripts/generate-ifc-fixtures.py`): `small-building.ifc`, `rotated-building.ifc` (+ `.json` expectations), plus **negative fixtures** `malformed-building.ifc` (bad STEP signature) and `unsupported-schema.ifc`.
- `fixtures/CONVENTIONS.md`: canonical meters, right-handed coordinates, normalization audit rules. `two-room-building.json` is the acceptance model with known dimensions and semantic relationships.
- `PHASE0_AUDIT.md` documents the normalization audit of existing model-loading paths.

## 3. IFC Import Pipeline (Phase 2 ‚Äî the new IFC parsing capability)

### 3.1 Python worker ‚Äî `blender-worker/ifc_worker/ifc_worker.py` (441 lines, new; IfcOpenShell 0.8.5 + NumPy 2.2.1)
Design principle: **fail-closed** ‚Äî any doubt raises `WorkerError`; nothing partial ever returns.

**Input validation:**
- File must exist, suffix `.ifc`, size 1 byte‚Äì50 MB (`IFC_WORKER_MAX_MB` env).
- STEP signature check: `ISO-10303-21;` must appear in the first 128 bytes.
- Schema whitelist: exactly `IFC2X3`, `IFC4`, `IFC4X3` ‚Äî anything else rejected with the schema named.
- Element cap: 100,000 (`IFC_WORKER_MAX_ELEMENTS`).
- Every element **must** have a `GlobalId`; a missing one aborts the import naming the entity.

**Unit intelligence (`get_unit_name`):**
- Uses `ifcopenshell.util.unit.calculate_unit_scale`; rejects non-finite/non-positive scales; recognizes m/cm/mm/ft/in/km (else `"custom"` with the raw factor). All downstream coordinates multiply by this scale ‚Üí canonical meters.

**Semantic sidecar (`inspect_ifc`):** per element ‚Äî
- `globalId`, `class` (IFC entity type), `name`, `description`
- `parentGlobalId` (spatial container) and `storeyName` when contained in an `IfcBuildingStorey`
- `placement` ‚Äî translation extracted from the full local placement matrix (`ifcopenshell.util.placement.get_local_placement`), unit-scaled
- `properties` and `quantities` split from psets: sets named `Qto_*` become quantities, and **quantity values scale dimensionally** ‚Äî names containing "area" √óscale¬≤, "volume" √óscale¬≥, else linear (`_quantity_factor`)
- `materials`, `hasGeometry` flag, per-element `warning`
- File-level: project/site/building names, storeys with unit-scaled elevations, `elementCount`, `entitiesByClass` histogram, `proxyCount` (IfcBuildingElementProxy count ‚Äî a modeling-quality signal), SHA-256 `sourceHash`, file size, converter version.
- `IfcSpace` entities are included alongside `IfcElement` so rooms stay first-class selectable semantics.

**Geometry conversion (`convert_ifc`):**
- `ifcopenshell.geom` iterator with `use-world-coords`, multi-core (up to 8), glTF serializer with `use-element-guids` so **mesh node names embed GlobalIds** (enables click-to-select in the browser).
- Hard timeout (`IFC_WORKER_TIMEOUT`, default 120 s) enforced inside the iteration loop.
- Any element whose geometry fails serialization aborts the whole conversion (fail-closed, count reported).
- **GLB repair pass** (`_repair_ifcopenshell_glb`): re-parses the serializer's binary container chunk-by-chunk, validates magic/version/JSON chunk, normalizes `asset.version = "2.0"`, injects missing default `scene`, re-packs with correct padding ‚Äî fixes IfcOpenShell serializer output quirks so strict loaders (three.js GLTFLoader, gltf-transform) accept it.
- Post-conversion `validate_glb`: meshes must exist; bounded VEC3 position accessors required; global GLB bounds + dimensions computed from accessor min/max and attached to the report; each sidecar element gets `hasGeometry` back-filled and a warning when it rendered nothing.

**CLI contract:** `inspect <in>` ¬∑ `convert <in> <out.glb> [--dump-sidecar <json>]` ¬∑ `export <bim.json> <out.ifc>` ¬∑ `validate-glb <in>` ‚Äî JSON on stdout, making it agent- and test-friendly. Python unit tests in `ifc_worker/tests/`.

### 3.2 Worker HTTP layer ‚Äî `blender-worker/server.js` (+93 lines)
- `POST /ifc/convert`, `POST /ifc/export` behind **worker shared-secret auth** (`requireWorkerAuth`, same `WORKER_SHARED_SECRET` scheme as the rig worker).
- Base64 input decoding with data-URL prefix guard; 1 byte‚Äì50 MB enforcement.
- **SHA-256 content cache**: converted GLB + sidecar cached in `pawsome3d-ifc-cache`; identical file re-imports return instantly with `cached: true`.
- **Concurrency guard**: max 2 simultaneous Python processes (`IFC_MAX_CONCURRENT`), busy ‚Üí immediate 422 "retry shortly" rather than queue blow-up; per-process kill-timeout; temp dirs `0o600` files, always cleaned in `finally`.
- Dockerfile updated to install `ifc_worker/requirements.txt`.

### 3.3 App server + client
- `server.ts` `POST /api/bim/import-ifc` (auth required) ‚Üí `blender_client.ts` `convertIfc()` ‚Üí worker; errors surface as 422 with the worker's precise message.
- Client `importIfc()` in `src/api.ts`; `BimModelBuilder` file-picker upload (`FileUp`), busy state "Importing IFC".

### 3.4 Imported-model UX (`ImportedScene`)
- GLB loaded with GLTFLoader; every mesh matched to its sidecar element by GlobalId-in-node-name; `userData.globalId` set for picking.
- **Click-to-select** any element ‚Üí inspector shows class, name, storey, placement, psets, quantities, materials.
- **Class filter** dropdown (dynamic list from sidecar classes) hides all other classes' meshes.
- **Category colors** toggle: recolors meshes by IFC class (`IfcWall` terracotta, `IfcSlab` grey, `IfcWindow` cyan translucent, etc.) vs. original materials.
- Storey list and file-level metadata (schema, unit, element count) shown from the sidecar.

## 4. Constrained BIM Authoring (Phase 3)

### 4.1 Domain model ‚Äî `src/bim/model.ts`
- **9 element types:** `wall, slab, roof, opening, door, window, space, column, beam` (`BIM_ELEMENT_TYPES`).
- `BimModel` = name/siteName/buildingName + `levels[]` (id, name, elevation) + `elements[]` (id, type, name, levelId, position `[x,y,z]` in meters, optional `end` for walls, width/depth/height/thickness, `hostId`/`openingId` relations, free-form `properties`).
- `EMPTY_BIM_MODEL` starts with a Ground Floor at elevation 0.

### 4.2 Metric snapping
- `snap(value, increment)` ‚Äî configurable increment (UI selector, default 0.1 m), guards non-finite/non-positive increments, rounds with 12-decimal float hygiene. Applied on element placement and wall endpoints.

### 4.3 Validation (`validateBimModel`) ‚Äî every rule returns a named error
- Levels: ‚â• 1, unique stable IDs, finite elevations.
- Elements: ‚â• 1, unique IDs, name required, type in whitelist, `levelId` must resolve.
- Positions: exactly three finite coordinates.
- **Walls** need a non-degenerate `end` point (no zero-length walls).
- **Openings** must host on an existing wall (`hostId` ‚Üí wall).
- **Doors/windows** must fill an existing opening (`openingId` ‚Üí opening) ‚Äî the IFC voids/fillings relationship enforced at author time.
- All provided dimensions must be positive and finite.

### 4.4 Pre-build report (`preflightBimModel`)
- `BimPreflightReport`: pass/fail + errors + **warnings** (no `space` ‚áí "room schedules will be empty"; no `slab`), level/element counts, and **intended physical bounds** computed from positions + dimensions + wall endpoints. This is the free gate and the dimension baseline the paid build is later verified against.

### 4.5 Undo/redo ‚Äî `bimHistoryReducer`
- Pure reducer over `{past, present, future}`; actions `replace / add-level / add-element / remove-element / undo / redo`; bounded history; drives the toolbar's Undo2/Redo2 buttons.

### 4.6 Authoring UX (`BimModelBuilder.tsx` + `AuthoredScene`)
- Palette buttons per element type with **smart defaults**: doors 2.1 m high, windows 1.2 m, walls 3 m √ó 0.2 m thick with an auto 4 m endpoint, slabs/roofs 4√ó3√ó0.2 m (roof auto-elevated to z = 3), new openings auto-host on the first wall, doors/windows auto-bind to the first opening.
- Live 3D viewport (R3F Canvas + Grid + OrbitControls): walls rendered as boxes rotated by `-atan2(dy,dx)` between endpoints; spaces at 18% opacity, openings 35%; selected element glows (emissive highlight); click-to-select; type filter dropdown shared with import mode.
- Element inspector for editing dimensions/positions; delete (Trash2); notes field; site/building naming.

## 5. Verified Paid Builds & Pricing

### 5.1 Two-gate verification doctrine
1. **Pre-build (free):** client calls `POST /api/bim/preflight` ‚Üí server runs `preflightBimModel`, returns report + mode + price (422 if failing). The server **re-runs preflight inside `/api/bim/build`** ‚Äî the UI's pass is never trusted, and no charge occurs before this gate.
2. **Post-build (paid):** the delivered artifact itself is parsed and verified; a failed post-build gate refunds the charge (`creditsDebited` tracking in the route).

### 5.2 Shell build ‚Äî `server/bim/shell.ts` (new, 60 credits)
- Pure gltf-transform `Document` authoring: one shared **unit-cube accessor pair** instanced per element via node scale/translation/quaternion (walls rotated from endpoints); per-type PBR colors, windows alpha-BLEND; openings/spaces excluded from geometry.
- **Post-build verification parses the delivered bytes, not the authoring document:** GLB re-read with NodeIO; mesh count must equal rendered elements; bounds finite; bounds must match the authoring scene's expected bounds within 1e-5 per axis; sane byte length. Any miss throws ‚Üí refund path.
- Returns `glb_base64` + verification manifest `{stage, passed, format: "glb-shell", meshCount, bounds, dimensionsPreserved, sizeBytes}`.

### 5.3 IFC/BIM build ‚Äî `export_bim` in the Python worker (300 credits)
- Authors a genuine **IFC4** file with `ifcopenshell.api`: IfcProject ‚Üí IfcSite ‚Üí IfcBuilding ‚Üí IfcBuildingStorey hierarchy (aggregation), SI units, Model/Body contexts.
- Element mapping: the 9 author types ‚Üí `IfcWall, IfcSlab, IfcRoof, IfcOpeningElement, IfcDoor, IfcWindow, IfcSpace, IfcColumn, IfcBeam`; spaces aggregated spatially, others contained in storeys.
- **Walls** built with `create_2pt_wall` (true parametric 2-point walls with elevation/height/thickness); all other types as parametric box solids (`_box_geometry`, dimensions clamped ‚â• 1 mm).
- **Relationships:** openings attached to host walls as voids (`add_feature`), doors/windows attached to openings as fillings (`add_filling`) ‚Äî real IFC semantics, not just geometry.
- **Property sets:** every element gets `Pset_Pawsome3D` carrying its authoring properties + `Pawsome3DId` (round-trip identity between author model and IFC).
- Client-supplied GlobalIds preserved when present; 10,000-element export cap.
- **Verification chain:** the exported IFC is *reopened and re-inspected with the same fail-closed inspector*, then converted to GLB. `server.ts` then asserts: schema == IFC4, exported element count == authored count == sidecar count, every element has a GlobalId, and **sorted rendered GLB dimensions match intended preflight dimensions within max(0.25 m, 2%)** ‚Äî catching axis-swap and scale bugs, tolerant of wall-thickness effects. Only then is the build delivered.

### 5.4 Pricing integration ‚Äî `src/pricing.ts`, `server.ts`
- `CREDIT_PRICES.BIM_SHELL_MODEL = 60`, `BIM_IFC_MODEL = 300`; `bimModelCost(mode)` shared client/server; price list entries ("GLB geometry without BIM semantics" / "IFC4 + semantic GLB + verification").
- Build route: admin bypass, `deductCredits` with reason `bim_{mode}_build`, 402 with exact price on insufficient credits, fresh balance returned, refund on post-build failure. Covered in `tests/pricing.test.mjs`.

### 5.5 Deliverables & downloads
- Mode toggle Shell vs IFC in the builder; downloads via `base64Download`: **GLB** (both modes), **IFC file + semantic sidecar JSON** (IFC mode). Pre-build and post-build verification reports rendered in the UI (`preflight` / `postBuild` state).

## 7. Operations, Security & Limits

- **Auth layers:** user JWT on all `/api/bim/*` routes; shared-secret on worker `/ifc/*` routes (same `WORKER_SHARED_SECRET` contract as the rig worker ‚Äî must match app + worker deployments).
- **Resource ceilings (env-tunable):** 50 MB IFC input, 120 s conversion timeout, 100k inspect elements, 10k export elements, 2 concurrent Python processes, 50 MB express body limit.
- **Hygiene:** SHA-256 content caching, mkdtemp + 0o600 files, guaranteed temp cleanup, `gc.collect()` after serializer teardown, kill-on-timeout.
- **Fail-closed philosophy end to end:** invalid input, unknown schema, missing GlobalId, geometry failure, or dimension drift never produces a partial deliverable ‚Äî and never charges (or refunds) the user.
- **Library decision (documented):** IfcOpenShell 0.8.5 + NumPy 2.2.1 server-side only; no `web-ifc` in the browser (bundle size + duplicate-parser semantics risk); three.js/R3F/gltf-transform reused for rendering and GLB validation.
